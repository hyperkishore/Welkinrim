Brand & Logo #

The WelkinRim logo is always displayed inverted (white) on dark backgrounds. Maintain clear space equal to the height of the "W" mark around the logo at all times.

Primary (dark background)
Original (light background)
On accent gradient

Do

Use the white inverted logo on dark backgrounds. Maintain minimum clear space. Keep aspect ratio locked.

Don't

Don't stretch, rotate, or add effects to the logo. Don't place it on busy backgrounds without a scrim. Don't use below 100px width.

Cards & Containers #

Cards use glass-morphism: semi-transparent backgrounds with subtle borders and optional backdrop-blur. The primary card pattern is used throughout for content blocks, testimonials, and feature highlights.

Glass Card

The standard card pattern. Semi-transparent background with 8% white border and 16px radius. Used for feature cards, testimonials, and content blocks.

background: rgba(255,255,255,0.03);
border: 1px solid rgba(255,255,255,0.08);
border-radius: 16px;
backdrop-filter: blur(20px);

Elevated Card

Slightly brighter variant for hover states or highlighted items. Used for active testimonial cards on mobile.

Modal Card

Opaque dark background for modals and popups. Dramatic shadow for depth separation from the overlay.

Form Elements #

Form inputs use dark backgrounds (#0a0a0f) with subtle borders. Focus state adds an amber glow ring. Labels are uppercase, small, and use 60% white opacity.

Interactive Effects

Scroll Reveal

Elements animate in as they enter the viewport using IntersectionObserver. Classes: .reveal, .reveal-up, .reveal-left, .reveal-right, .reveal-scale

Card Tilt (3D)

Value and testimonial cards rotate subtly on mouse hover using perspective transforms. Disabled on mobile and reduced-motion.

Magnetic Buttons

Primary and large buttons follow the cursor slightly on hover for a magnetic feel. Disabled on mobile and reduced-motion.

Usage Guidelines #

Key rules for maintaining brand consistency across all WelkinRim pages and materials.

Do

Use the dark theme consistently. All pages use #0a0a0f or #000 body backgrounds. Keep text at 90% white for body, 100% for headings.

Don't

Don't mix light and dark sections. The site is fully dark since v1.2.24. Don't use any .section-light backgrounds on new pages.

Do

Use shared.js for nav, footer, WhatsApp button, quote modal, sticky bar, and exit popup. Every page gets these automatically.

Don't

Don't duplicate nav or footer HTML in pages. Don't hardcode API keys in source files. Don't skip theme-color and color-scheme meta tags.

Do

Use CSS custom properties (--color-accent, --radius-lg, etc.) for all values. Include OG meta tags on every page. Add pages to sitemap.xml.

Don't

Don't use hardcoded hex values when a CSS variable exists. Don't create new font stacks. Don't skip canonical URLs or structured data schemas.

<meta name="color-scheme" content="dark">
